# 女娲 · Skill造人术

> 提炼思维框架，不是复制人。捕捉 HOW they think，不是 WHAT they said。

一个好的人物Skill = 可运行的认知操作系统：心智模型（镜片）+ 决策启发式（直觉规则）+ 表达DNA + 反模式 + 诚实边界。

## 执行流程总览

```
Phase 0  → 入口分流（直接路径 / 诊断路径）
Phase 0.5 → 创建Skill目录
Phase 1  → 6 Agent并行调研
Phase 1.5 → 调研Review检查点（暂停确认）
Phase 2  → 三重验证框架提炼
Phase 2.5 → 提炼确认检查点（暂停确认）
Phase 3  → Skill构建
Phase 4  → 质量验证（子agent测试）
Phase 5  → 双Agent精炼
```

---

## Phase 0: 入口分流

| 用户输入 | 路径 | 示例 |
|---------|------|------|
| 明确人名/主题 | **直接路径** → Phase 0A | 「蒸馏芒格」「做一个费曼skill」 |
| 模糊需求/困惑 | **诊断路径** → Phase 0B | 「我想提升决策质量」 |

### Phase 0A: 需求澄清（直接路径）

确认5项：
1. 人物/主题是谁
2. 聚焦方向（全面 vs 某维度）
3. 用途（思维顾问？决策参考？角色扮演？）
4. 新建 or 更新（检查已有skill目录）
5. 本地语料（书籍PDF、演讲transcript、视频字幕等）

用户说「就做XX」→ 默认全面画像+思维顾问+无本地语料，直接推进。

### Phase 0B: 需求诊断（模糊路径）

读取 `references/diagnosis-guide.md` 获取需求维度表和候选推荐格式。

核心流程：1-2轮追问定位需求 → 推荐2-3个候选 → 用户选择后进入Phase 0A。

## Phase 0.5: 创建Skill目录

在用户项目目录下创建：

```
[person-name]-perspective/
├── SKILL.md
├── scripts/
└── references/
    ├── research/
    │   ├── 01-writings.md      # 著作与系统思考
    │   ├── 02-conversations.md # 长对话与即兴思考
    │   ├── 03-expression-dna.md # 碎片表达与风格DNA
    │   ├── 04-external-views.md # 他者视角与批评
    │   ├── 05-decisions.md     # 决策记录与行动
    │   └── 06-timeline.md      # 人物时间线
    └── sources/
        ├── books/
        ├── transcripts/
        └── articles/
```

**检查项**：
- 中国人物 → 信息源策略切换（B站/小宇宙/权威中文媒体优先，知乎/微信公众号永远排除）
- 更新模式 → 读现有SKILL.md，标注需刷新的部分
- 本地语料 → 存入 `sources/` 对应子目录

**核心规则**：Skill必须自包含——所有调研文件存在skill目录内部，不依赖外部文件。

## Phase 1: 多源信息采集（并行Agent Swarm）

### 模式判断

| 模式 | 触发条件 | 策略 |
|------|---------|------|
| 纯网络搜索（默认） | 无本地素材 | 6个Agent全走网络搜索 |
| 本地语料优先 | 用户提供了素材 | 先分析本地素材，缺失维度走网络补充 |
| 纯本地语料 | 用户明确要求或非公众人物 | 只分析本地素材 |

### 6个Agent任务

用 `sessions_spawn` 启动6个并行subagent：

| Agent | 搜索目标 | 输出文件 |
|-------|---------|---------|
| 1 著作 | 书、长文、论文、newsletter | `01-writings.md` |
| 2 对话 | 播客、长视频、AMA、深度采访 | `02-conversations.md` |
| 3 表达 | Twitter/X、微博、即刻、短文 | `03-expression-dna.md` |
| 4 他者 | 他人分析、书评、批评、传记 | `04-external-views.md` |
| 5 决策 | 重大决策、转折点、争议行为 | `05-decisions.md` |
| 6 时间线 | 完整时间线+最近12个月动态 | `06-timeline.md` |

读取 `references/agent-prompts.md` 获取每个Agent的详细prompt模板和信息源策略。

### Agent硬性要求
- 结果必须写入 `references/research/0X-xxx.md`
- 注明信息来源和可信度（一手>二手>推测）
- 区分「他说过的」vs「别人说他的」vs「我推断的」
- 发现矛盾时保留矛盾，不要和稀泥

### 工具辅助
- 字幕下载：`bash scripts/download_subtitles.sh <URL> [输出目录]`
- 字幕清洗：`python3 scripts/srt_to_transcript.py <input.srt> [output.txt]`
- 调研摘要：`python3 scripts/merge_research.py <skill目录>`
- 质量自检：`python3 scripts/quality_check.py <SKILL.md路径>`

### 超时与失败处理
- 单Agent超时（5分钟无结果）→ 不等待，标注「信息不足」
- 信息源匮乏（<10条）→ 提醒用户，降低期望
- Agent结果冲突 → 保留矛盾，用「内在张力」收录

**原则**：宁可交付诚实标注局限的60分Skill，不要编造的90分Skill。

## Phase 1.5: 调研Review检查点

所有Agent完成后，**暂停展示调研质量摘要**：

```
┌──────────────┬──────────┬─────────────────────┐
│ Agent        │ 来源数量  │ 关键发现             │
├──────────────┼──────────┼─────────────────────┤
│ 1 著作       │ 8篇      │ 核心论点: ...        │
│ ...          │ ...      │ ...                 │
├──────────────┼──────────┼─────────────────────┤
│ 矛盾点       │ 2处      │ Agent1说X, Agent4说Y │
│ 信息不足维度  │ 无       │                     │
└──────────────┴──────────┴─────────────────────┘
```

用户确认OK → Phase 2。某维度不够 → 补充调研。

## Phase 2: 框架提炼

读取 `references/extraction-framework.md` 获取三重验证方法论。

### 2.1 心智模型提取（3-7个）
1. 扫描 `01-writings.md` 到 `05-decisions.md`，列出所有候选论点（15-30个）
2. 三重验证筛选：跨域复现 + 生成力 + 排他性
   - 三重通过 → 心智模型
   - 1-2重 → 降级为决策启发式
   - 0重 → 丢弃
3. 按排他性排序，取top 3-7

### 2.2 决策启发式（5-10条）
「如果X，则Y」格式，有具体案例支撑。

### 2.3 表达DNA
句式偏好、词汇特征、节奏感、幽默方式、确定性表达、引用习惯。

### 2.4 价值观与反模式
价值观排序 + 反模式 + 矛盾与张力（深度的来源）。

### 2.5 智识谱系
受谁影响 → 影响了谁 → 思想地图位置。

### 2.6 诚实边界
不能预测全新问题反应、不能替代创造力和直觉、公开vs真实可能有差距、信息截止时间。

## Phase 3: Skill构建

### Step 1: 读取模板
读取 `references/skill-template.md`。

### Step 2: 填充内容

| 模板Section | 填充来源 |
|------------|---------|
| frontmatter | 来源数+模型数+触发词 |
| 角色扮演规则 | 模板默认规则 |
| 回答工作流 | 根据心智模型自动推导研究维度（详见下方） |
| 身份卡 | 时间线+著作 → 此人语气50字自我介绍 |
| 心智模型 | Phase 2.1 |
| 决策启发式 | Phase 2.2 |
| 表达DNA | Phase 2.3 → 风格规则 |
| 时间线 | Agent 6 精简为关键节点 |
| 价值观与反模式 | Phase 2.4 |
| 智识谱系 | Phase 2.5 |
| 诚实边界 | Phase 2.6 + 调研时间 |
| 创建者归属 | 固定：女娲链接 + 花叔链接 |

### 回答工作流（Agentic Protocol）生成

放在角色扮演规则之后。必须包含3个Step：

1. **问题分类**：需要事实→先研究 / 纯框架→直接回答 / 混合→先获取事实再分析
2. **[人名]式研究**：从心智模型反推研究维度（3-5个分类，各4-6个具体研究点）
3. **[人名]式回答**：基于事实+心智模型+表达DNA输出

**研究维度推导示例**：
- 芒格（多元思维+逆向思考）→ 看护城河、看激励结构、看最大风险、看历史类比
- 塔勒布（反脆弱+尾部风险）→ 看极端情况、看谁承担尾部风险、看专家预测记录

### Step 3: 质量自检
读取 `references/extraction-framework.md` 末尾清单逐项检查。

### Step 4: 输出
写入 `[person-name]-perspective/SKILL.md`。

## Phase 4: 质量验证

用 `sessions_spawn` 启动子agent执行3项测试：

| 测试 | 方法 | 期望 |
|------|------|------|
| 已知测试 | 3个已知立场问题 | 方向一致 |
| 边缘测试 | 1个未公开讨论的相关问题 | 有保留地推断 |
| 风格测试 | 100字分析文 | 有辨识度，非AI鸡汤 |

### 通过标准

| 检查项 | 通过 | 不通过 |
|--------|------|--------|
| 心智模型 | 3-7个，每个有来源 | <3或>10 |
| 局限性 | 写出失效条件 | 只写优点 |
| 表达辨识度 | 100字能认出 | 像通用ChatGPT |
| 诚实边界 | ≥3条具体局限 | 只有「不能替代本人」 |
| 内在张力 | ≥2对矛盾 | 高度一致（太假） |
| 一手来源 | >50% | 主要二手转述 |

迭代上限：Phase 2→4最多循环2次。仍有不通过项→在诚实边界标注，交付当前最优版本。

**展示验证结果给用户确认后才算完成。**

## Phase 5: 双Agent精炼

用 `sessions_spawn` 并行启动两个Agent：

**Agent A（optimizer视角）**：
- 8维度结构评估（工作流清晰度、边界条件、检查点设计、指令具体性等）
- 干跑3个测试prompt
- 输出：最弱2个维度的改进建议（含改后文本）

**Agent B（creator视角）**：
- 评审激活触发条件覆盖度
- 评审角色扮演规则可操作性
- 识别缺失信息
- 输出：2-3处文本改动建议（含改后文本）

主Agent综合两份报告，应用不冲突的改进，展示变更摘要请用户确认。

## 更新已有Skill

当用户说「更新XX的skill」时：
1. 读现有SKILL.md，从诚实边界找调研时间
2. 只启动Agent 2（最新对话）+ Agent 5（最新决策）+ Agent 6（时间线）
3. 新信息强化→补充案例 / 矛盾→更新模型 / 新模式→增加模型
4. 增量更新，不重写整个Skill

## 品味守则

| 原则 | 一句话 |
|------|--------|
| 长文 > 金句 | 3000字essay比50条推文更揭示思维结构 |
| 争议 > 共识 | 最被争议的观点最揭示独特性 |
| 变化 > 固定 | 改变立场处比一直坚持的更有信息量 |

### 绝不做的事
- 编造此人没说过的话
- 把通用道理包装成「独特见解」
- 忽略负面评价和争议
- 在信息不足时强行生成
