# 特殊场景处理指引

## 活人 vs 历史人物

- **活人**：注意时效性，标注截止日期，建议定期更新
- **历史人物**：材料更稳定但可能有传记偏差，多源交叉验证

## 主题Skill vs 人物Skill

输入是主题（如「价值投资」「反脆弱决策」）时，各Phase变体：

| Phase | 人物Skill | 主题Skill变体 |
|-------|----------|--------------|
| 0A | 确认人名+聚焦方向 | 确认主题边界+目标受众 |
| 0.5 | `[person]-perspective/` | `[topic]-framework/` |
| 1 | 6个Agent围绕一个人 | 先搜3-5个核心人物/流派，按人物分配Agent |
| 2.1 | 提取一个人的心智模型 | 提取**领域共识框架** + **各家分歧** |
| 2.3 | 模拟一个人的表达 | 中性但专业的表达，不模拟特定人物 |
| 2.4 | 一个人的内在矛盾 | 流派间的根本分歧 |
| 3 | 用 skill-template.md | 去角色扮演和身份卡，改为「框架概览」+「流派对比」 |
| 4 | 对比此人已知立场 | 对比领域内公认经典案例 |

## 中国人物 vs 西方人物

- **中国人物**：B站原始视频/演讲、小宇宙播客、权威媒体（36氪/晚点/财新/极客公园）、本人著作/微博
- **西方人物**：Twitter、YouTube、Podcast、Amazon书评
- **永远排除**：知乎、微信公众号、百度百科

## 冷门人物（公开信息极少）

当可用来源<10条时：
1. Phase 0.5就告知用户信息受限
2. 心智模型减至2-3个，标注「基于有限信息推测」
3. 诚实边界加大篇幅，列出信息不足的维度
4. 引导用户提供一手素材

## 蒸馏用户自己

1. 无法从公开渠道搜到，需用户提供素材
2. 引导提供：个人文章/博客、视频/播客、决策备忘录、自我描述
3. 6个Agent改为分析用户素材而非网络搜索
4. 注意「自我认知偏差」——可追问身边人的评价
