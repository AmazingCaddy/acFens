# Lessons 📝

## 输出到公开仓库前必须过滤敏感信息
- **来源：** 2026-04-06 daily PR 包含 chat_id
- **教训：** 写入 acFens 的内容会推到 GitHub，任何 ID、token、secret、chat_id 都不能出现。写完后要 grep 自检。
- **次数：** 1

## 频繁重启 gateway 会丢消息
- **来源：** 2026-04-06 飞书配置调试
- **教训：** 重启 gateway 导致 WebSocket 断连，用户消息丢失。非必要不重启，改配置后一次性重启就够了。
- **次数：** 1

## 隔离 cron session 工具受限
- **来源：** 2026-04-06 日报权限问题
- **教训：** isolated session 的工具列表和主 session 不同，feishu_perm 等工具不可用。需要用 curl 等替代方案绕过。
- **次数：** 1

## 不能凭假设下结论
- **来源：** 2026-04-05 Copilot 授权账号确认
- **教训：** auth-profiles.json 只存了 token 没存用户名，我却声称知道是哪个账号。没有证据的结论不是结论。
- **次数：** 1

## cherry-pick 合并冲突要逐行检查
- **来源：** 2026-04-04 迁移分支操作
- **教训：** 用 --strategy-option theirs 解冲突时，丢失了第10条原则。自动合并策略不等于正确，合并后必须 diff 检查完整性。
- **次数：** 1

## 安全检查是我的责任
- **来源：** 2026-04-04 openclaw-setup-guide.md 事件
- **教训：** 第一版包含真实邮箱，被老王拦截。输出前必须自检敏感信息，不依赖用户提醒。
- **次数：** 1

## 不要轻易说"不支持"
- **来源：** 2026-04-04 飞书事件
- **教训：** 老王问能不能接飞书，我说不支持。结果他自己搞定了。以后先调查确认，不要凭印象下结论。
- **次数：** 1
