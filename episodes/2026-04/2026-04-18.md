## frp 内网穿透搭建
- 目的：从外网 SSH 回家里的 Mac，远程操作 Claude Code
- 在 Azure VM（跑 OpenClaw 的这台）上部署了 frps 服务端
  - frps v0.61.1，systemd 自启，端口 7000
  - 带 auth.token 认证
  - Azure NSG 开放 7000、6000 端口
- Mac 上用 Docker 方式跑 frpc 客户端
  - 配置文件放在 `/Users/<YOUR_USER>/dockerVolumn/frp/frpc.toml`
  - `--restart=always` 自动重启
  - 踩坑：Docker 容器里 `127.0.0.1` 不是宿主机，要用 `host.docker.internal`
- 连接方式：`ssh -p 6000 <YOUR_USER>@<VM_IP>`
- 搭建过程记录到 `acFens/knowledge/technical/frp-setup-guide.md`（已脱敏）

### 踩坑
- brew install frpc v0.68.1 的 bottle 二进制损坏（才 92.9KB），实际是 Microsoft Defender 拦截导致
- 手动下载 frpc 同样被 Defender 拦截，最终用 Docker 方案绕过
- Docker 里 localhost 指向容器自己，必须用 `host.docker.internal`

## SSH 密钥认证
- iPad 上用 Termius 生成 ED25519 密钥
- 公钥写入 Mac 的 `~/.ssh/authorized_keys`
- 关闭密码登录
  - 踩坑：macOS 的 `/etc/ssh/sshd_config` 修改不生效，因为 `sshd_config.d/` 下的配置优先级更高
  - 解决：在 `/etc/ssh/sshd_config.d/200-no-password.conf` 中写入配置
- Termius 的 Sign in with Apple 在国内可能有问题（一直转圈），但跳过登录也能用

## Auth0 讨论
- Native Application 支持 web 的 redirect URI
- 迁移场景：纯 Native → Chromium + WebUI 混合应用
- 同一个 Client ID 可以同时支持 custom scheme 和 https callback，用 PKCE 统一
- Allowed Logout URLs 是给 `/v2/logout` 跳转用的，纯 Native 不调 logout endpoint 则不需要配

## 教训
- macOS sshd 配置有 include 机制，直接改 sshd_config 可能被覆盖，要注意 sshd_config.d/ 目录
- brew 的 bottle 不一定靠谱，遇到二进制损坏时直接从 GitHub releases 手动下载
